## 一、项目背景与目标

### 1.1 需求来源

根据参考视频（心理疗愈类学习 App），提炼核心用户路径：**在旅程地图中浏览课程进度 → 进入课程学习长文内容 → 完成反馈 → 解锁下一节点**。本 Demo 聚焦该主流程，验证全栈架构设计、前后端协作与国际化能力。

### 1.2 项目目标

构建一个**可运行、可打包、可扩展**的 Demo，满足以下目标：

1. 前后端分离，接口契约清晰，便于多人并行开发
2. 数据库支持章节、课程、进度、多语言文案
3. 前端还原参考视频的核心交互与视觉风格
4. 课程正文超过 5000 字，支持富文本排版与分段揭示阅读
5. 前后端均实现国际化（默认 `zh-CN`，可切换 `en-US`）

### 1.3 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| 前端 | React Native (Expo SDK 52) + TypeScript + expo-router | 符合岗位要求，一套代码打包 Web / Android |
| 后端 | Node.js + Express + Prisma | 与前端同语言栈，Prisma 简化 ORM 与迁移 |
| 数据库 | SQLite | Demo 轻量部署，可平滑迁移 PostgreSQL |
| 国际化 | i18next（前端）+ Translation 表（后端） | 前后端文案分离，符合内容国际化要求 |
| 部署 | Docker（后端）+ expo export / EAS Build | 覆盖服务端部署与移动端打包 |

## 二、功能拆解与验收标准

### 功能 1：底部 Tab 导航

| 项 | 说明 |
|----|------|
| **目标** | 实现 5 个 Tab：今日 / 旅程 / 探索 / 陪伴 / 个人资料，默认进入「旅程」；语言切换后 Tab 文案同步 |
| **验收方式** | 运行后底部导航可切换；非核心 Tab 显示占位文案，不影响主流程 |

### 功能 2：课程地图（旅程页）

| 项 | 说明 |
|----|------|
| **目标** | 生成可上下滚动的课程地图，展示章节信息、节点状态（可学 / 已完成 / 锁定）、星星积分；两章共 14 个节点（各 7 个），S 形连续蜿蜒路径 |
| **验收方式** | 可见 Chapter 标签、金色可学节点带右上角小范围闪烁星星簇；锁定节点呈灰色锁态；整条路径为连续 `translateX` 偏移，非左/中/右三档硬跳 |

### 功能 3：章节切换与详情

| 项 | 说明 |
|----|------|
| **目标** | ① 主页面纵向滚动时，根据章节锚点自动切换顶部章节头（第 1 章 → 第 2 章）；② 点击章节头展开 Bottom Sheet，展示当前章节介绍与探索要点；③ 第 2 章第一个节点前有「思维 / Thinking」分隔线 |
| **验收方式** | 滑过第 1 章末节点后，顶部立即切到「第 2 章 / 思维」；Sheet 弹出流畅；第 2 章路径起始处可见分隔线；第 1 章无中段分隔线 |

### 功能 4：节点交互弹窗

| 项 | 说明 |
|----|------|
| **目标** | 点击可学节点弹出课程卡片，展示标题、类型（Reading / Practice）、操作按钮 |
| **验收方式** | 锁定节点点击无响应；可学节点弹出 Start；已完成节点显示 Continue |

### 功能 5：课程学习全流程

| 项 | 说明 |
|----|------|
| **目标** | 封面页 → 正文分段阅读（Continue 加载下一段）→ 完成页（表情评分） |
| **验收方式** | 正文按段自上而下展开；点 Continue 时在**已显示正文底部**出现加载菊花（非全屏遮罩），随后新段落 reveal；进度条按已揭示段落比例推进；Aa 可调字号；完成后返回地图并更新节点 |

### 功能 6：进度解锁与积分

| 项 | 说明 |
|----|------|
| **目标** | 完成课程后获得星星奖励，自动解锁同章节下一节课（章末解锁下一章第 1 课） |
| **验收方式** | 完成第 1 节后星星 +1，第 2 节由锁定变为可学；当前可学节点带闪烁星星簇 |

### 功能 7：国际化

| 项 | 说明 |
|----|------|
| **目标** | 支持 `zh-CN` / `en-US`；默认中文；前端 UI 文案与后端课程数据均按语言返回；切换后旅程页重置并重新拉取 |
| **验收方式** | 在 Profile 切换语言后，Tab 标签、按钮、章节标题、课程标题、正文同步切换，无中英混排 |

### 功能 8：长文富文本排版

| 项 | 说明 |
|----|------|
| **目标** | 第一节课程正文超过 5000 字，包含标题、段落、有序 / 无序列表；前端按 HTML 区块分页揭示 |
| **验收方式** | API 返回 `contentLength > 5000`；前端正确渲染 HTML；Continue 逐段追加显示 |

## 三、数据库表结构设计

采用 **主表 + Translation 表** 模式，将结构与多语言文案解耦，便于后续扩展更多语言。

```prisma
model Chapter {
  id           Int                  @id @default(autoincrement())
  sortOrder    Int
  slug         String               @unique
  translations ChapterTranslation[]
  lessons      Lesson[]
}

model ChapterTranslation {
  id          Int     @id @default(autoincrement())
  chapterId   Int
  locale      String              // "en-US" | "zh-CN"
  title       String              // e.g. "Introduction" / "入门"
  label       String              // e.g. "CHAPTER 1" / "第 1 章"
  description String?
  bullets     String              // JSON: ["...", "..."]
  @@unique([chapterId, locale])
}

model Lesson {
  id          Int    @id @default(autoincrement())
  chapterId   Int
  sortOrder   Int
  type        String               // "reading" | "practice"
  durationMin Int    @default(3)
  starReward  Int    @default(1)
  translations LessonTranslation[]
  progress     UserLessonProgress[]
}

model LessonTranslation {
  id       Int    @id @default(autoincrement())
  lessonId Int
  locale   String
  title    String
  subtitle String?
  content  String                 // HTML, 首课 >5000 字
  @@unique([lessonId, locale])
}

model UserProfile {
  id       Int    @id @default(1)
  stars    Int    @default(0)
  locale   String @default("zh-CN")
  progress UserLessonProgress[]
}

model UserLessonProgress {
  id          Int       @id @default(autoincrement())
  userId      Int       @default(1)
  lessonId    Int
  status      String                 // locked | available | completed
  rating      Int?
  completedAt DateTime?
  @@unique([userId, lessonId])
}
```

### 种子数据规划

| 章节 | slug | 课程数 | 说明 |
|------|------|--------|------|
| Chapter 1 · Introduction / 入门 | `introduction` | **7** | 第 1 节默认 available，含 5000+ 字长文；含 Reading + Practice |
| Chapter 2 · Thinking / 思维 | `thinking` | **7** | 初始全部 locked；路径开头有「思维」分隔线 |

合计 **14** 个节点。种子变更后须执行 `npm run db:reset`。API 启动时会打印各章课程数量，便于发现脏库。

### 解锁规则

1. 初始状态：Chapter 1 第 1 节 `available`，其余 `locked`
2. 完成当前课 → 同章节 `sortOrder + 1` 的课程变为 `available`
3. 章节最后一课完成 → 下一章节第 1 课变为 `available`
4. 每完成一课，`stars += starReward`（默认 1）

## 四、前端设计思路

### 4.1 架构分层

```
┌─────────────────────────────────────┐
│  Screens (expo-router)              │
│  (tabs) / lesson/[id]/*            │
├─────────────────────────────────────┤
│  Components (journey / lesson)      │
├─────────────────────────────────────┤
│  Services (api.ts) + Context (i18n) │
├─────────────────────────────────────┤
│  Utils (pathLayout, locale)         │
│  Constants (theme, assets)          │
└─────────────────────────────────────┘
```

### 4.2 目录结构

```
spark-demo-app/
├── app/
│   ├── (tabs)/
│   │   ├── journey.tsx         # 核心：旅程地图 + 滚动切换章节
│   │   └── profile.tsx         # 语言切换
│   ├── lesson/[id]/
│   │   ├── intro.tsx           # 课程封面
│   │   ├── read.tsx            # 正文分段阅读
│   │   └── complete.tsx        # 完成评分
│   └── _layout.tsx
├── components/
│   ├── journey/
│   │   ├── LessonNode.tsx      # 节点状态 + 当前节点星星簇动画
│   │   └── JourneyComponents.tsx
│   └── lesson/
│       ├── RevealContent.tsx   # 段落自上而下展开
│       └── LessonComponents.tsx
├── utils/
│   ├── pathLayout.ts           # S 形路径连续 translateX
│   └── locale.ts               # normalizeLocale + 正文分段
├── services/api.ts             # cache: 'no-store' 防语言缓存
├── context/AppContext.tsx
├── i18n/index.ts
└── assets/images/
```

### 4.3 关键 UI 实现方案

| 模块 | 实现方案 | 说明 |
|------|----------|------|
| S 形蜿蜒路径 | `pathLayout.ts` 提供像素级 `translateX` 数组，按屏宽缩放 | 连续偏移：中→右→最右→回中→左→最左…，非离散左/中/右 |
| 章节滚动切换 | ScrollView 内每章 1px 锚点 + 固定 `SCROLL_SWITCH_OFFSET=96` | 锚点在章节间隙之后、首节点之前，保证 7/7 边界准确 |
| 章节分隔线 | API `sections[].beforeLessonOrder`；仅第 2 章首节点前展示 | 第 1 章无中段分隔，避免误读为「只有 4 个节点」 |
| 当前节点动画 | 右上角小范围四角星 + 大小不一圆点，自转 + 脉冲 | 对齐参考 demo 的闪烁星簇，非整圈公转 |
| 节点状态 | 组合 PNG（base / inner / outer / icon） | 官方 light 资源包 |
| 章节 Sheet | `Modal` 展示**当前滚动章节**详情 | 主流程靠滚动切章，Sheet 不负责跨章翻页 |
| 分段阅读 | `splitContentPages` + `RevealContent` 高度动画展开 | Continue → loading（菊花在正文底部）→ expanding |
| 富文本 | `react-native-render-html` | 解析后端 HTML |
| 阅读进度 | `revealedCount / sections.length` | 按段推进绿色进度条 |
| 字号调节 | Context `fontScale`（sm / md / lg） | Aa 循环切换 |
| 国际化 | i18next + API `locale` 同步；旅程页 `key={locale}` 强制刷新 | 默认 zh-CN |

### 4.4 状态与数据流

1. 进入旅程页 → `GET /journey` 拉取两章全部节点 + 进度
2. 滚动地图 → 比较锚点 Y 与 `scrollY + 96`，更新 `activeChapterId` 与章节头
3. 点击节点 → `status !== 'locked'` → 弹窗 → `/lesson/:id/intro`
4. 阅读 Continue → 本地分段揭示；最后一段 Complete → `POST /lessons/:id/complete`
5. `useFocusEffect` 回地图刷新；切换语言时清空锚点并重新 `load`

## 五、后端设计思路

### 5.1 架构分层

```
Request → locale middleware → Router → Service → Prisma → SQLite
```

### 5.2 目录结构

```
spark-demo-api/
├── src/
│   ├── app.ts                      # Express 入口；启动时打印各章课时数
│   ├── routes/index.ts
│   ├── services/journeyService.ts  # 旅程 / 解锁 / sections

│   ├── schema.prisma
│   └── seed.ts                     # 两章各 7 课 + 长文种子
```

### 5.3 核心业务逻辑

| 方法 | 职责 |
|------|------|
| `getJourney(locale)` | 聚合章节、课程、进度；Chapter 2 注入 `sections: [{ title, beforeLessonOrder: 1 }]` |
| `getLessonDetail(id, locale)` | 返回课程详情与 HTML 正文 + `contentLength` |
| `completeLesson(id, rating)` | 更新进度、累加星星、解锁下一课 |
| `pickTranslation(items, locale)` | 按 locale 取文案，fallback `en-US` |

### 5.4 语言解析策略

优先级：`?locale=` 查询参数 > `Accept-Language` Header > 默认 **`zh-CN`**

### 6.2 GET `/journey`

获取完整旅程地图数据（两章各 7 课，合计 14 节点）。

**Query**：`locale=zh-CN` 或 `locale=en-US`

**Response 示例（节选）**：

```json
{
  "code": 0,
  "data": {
    "chapters": [
      {
        "id": 1,
        "slug": "introduction",
        "label": "第 1 章",
        "title": "入门",
        "bullets": ["是什么真正把你带到这里", "..."],
        "lessons": [
          {
            "id": 1,
            "sortOrder": 1,
            "type": "reading",
            "title": "欢迎踏上你的个性化创伤疗愈之旅",
            "durationMin": 3,
            "status": "available",
            "starReward": 1
          }
        ],
        "sections": []
      },
      {
        "id": 2,
        "slug": "thinking",
        "label": "第 2 章",
        "title": "思维",
        "lessons": [ "…共 7 个 locked/available 节点…" ],
        "sections": [
          { "title": "思维", "beforeLessonOrder": 1 }
        ]
      }
    ],
    "activeChapterId": 1,
    "stars": 0,
    "locale": "zh-CN"
  }
}
```

> `sections` 语义：`beforeLessonOrder` = 在该 sortOrder 节点**之前**渲染分隔线；`afterLessonOrder` 为可选，表示节点之后。当前仅第 2 章使用 `beforeLessonOrder: 1`。

## 九、打包与部署

| 目标平台 | 命令 | 输出 |
|----------|------|------|
| 后端（Docker） | `docker build -t spark-demo-api . && docker run -p 3000:3000` | 容器化 API 服务 |
| Web | `cd spark-demo-app && npm run export:web` | `dist/` 静态文件 |
| Android APK（本地） | 见下方「本地打包 APK」 | [`docs/spark-demo.apk`](./spark-demo.apk) |
| Android APK（EAS 云端） | `eas login` 后 `eas build -p android --profile preview` | Expo 云端产物链接 |

**本地打包 APK**（本机需已安装 Android SDK；当前仓库已附带可安装包）：

```bash
cd spark-demo-app
npx expo prebuild --platform android
cd android && ./gradlew assembleRelease
# 产物：android/app/build/outputs/apk/release/app-release.apk
# 交付副本：docs/spark-demo.apk
```

模拟器访问本机后端默认 `http://10.0.2.2:3000/api/v1`；真机请设置局域网 IP，例如在 `app.json` 增加：

```json
"extra": { "apiUrl": "http://192.168.x.x:3000/api/v1" }
```

然后重新打包。Release 已开启 `usesCleartextTraffic`，允许 HTTP 联调。

本地开发务必先重置数据库：

```bash
cd spark-demo-api
PRISMA_USER_CONSENT_FOR_DANGEROUS_AI_ACTION='可以' npm run db:reset
npm run dev
```

启动日志应显示：`Chapter 1: 7 lessons`、`Chapter 2: 7 lessons`。

## 十一、迭代对照（相对初版计划）

| 变更点 | 初版计划 | 当前实现 |
